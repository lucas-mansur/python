# Exercícios - Python Impressionador (Hashtag Treinamentos)

Esta pasta contém as resoluções e estudos dos exercícios práticos realizados ao longo do curso **Python Impressionador**.

## 📂 Estrutura de Pastas

Para manter a organização alfabética correta e o alinhamento com a ordem dos módulos oficiais do curso, os arquivos estão distribuídos em subpastas numeradas com dois dígitos:

*   **`04. Seus primeiros programas em Python`**: Fundamentos de variáveis, entradas (`input`), saídas (`print`) e operações matemáticas sequenciais.
*   **`06. If - Condições em Python`**: Estruturas de decisão lógicas, blocos condicionais (`if`, `elif`, `else`).
*   **`07. Strings - Textos e a importância no Python`**: Métodos de strings, fatiamentos, formatações e tratamento de textos.
*   **`08. Listas Python - Métodos e Usos`**: Criação, modificação, fatiamento, loops e métodos integrados para manipulação de listas.
*   **`09. For - Estrutura de Repetição`**: Loops baseados em controle e iterações definidas (`for`).
*   **`10. While e Criando um Loop infinito`**: Estrutura de repetição baseada em condições lógicas (`while`) e tratamento de loops infinitos.
*   **`11. Tuplas - Uma lista muito útil e imutável`**: Conceito de coleções ordenadas imutáveis e aplicações práticas de tuplas.
*   **`12. Dicionários em Python`**: Estruturas de dados chave-valor, cartilhas de métodos e exercícios avançados de mapeamento.

## 💡 Diretrizes para Novos Exercícios

Para manter este diretório sempre organizado, siga este padrão quando avançar para novos módulos:

1. **Crie uma nova pasta** utilizando o número de dois dígitos e o nome correspondente do módulo do curso. Exemplo: `14. Criando suas funções em Python`.
2. **Trabalhe preferencialmente com arquivos `.ipynb`** (Jupyter Notebook), pois eles permitem incluir textos descritivos formatados, fórmulas e gráficos de maneira muito mais didática.
3. Se você precisar exportar um script `.py` para executar fora do Notebook, tente mantê-lo apenas se for necessário para um deploy ou script final de automação. Caso contrário, opte por manter apenas o `.ipynb` para evitar arquivos redundantes.

## ⚡ Dicas de Estudo

1. **Faça anotações nas células de Markdown:** Sempre que aprender uma função nova ou um comportamento interessante de um erro, documente diretamente na célula acima do código no notebook.
2. **Crie o hábito do Git:** Sempre que terminar um lote de exercícios de um módulo, faça um commit das suas alterações. Isso ajuda a registrar seu progresso ao longo do tempo.
3. **Não copie soluções:** Tente quebrar a cabeça antes de olhar o gabarito. Se não conseguir, estude o gabarito, feche-o e escreva a sua própria lógica do zero.
